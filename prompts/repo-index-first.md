# Repo index first (token discipline)

> Before any substantive work in this repo, produce or refresh
> `agent/INDEX.md` (≤ 150 lines): purpose in two sentences; directory map
> with one line per top-level dir; entry points; build/test/verify commands
> verbatim; where ground truth lives; conventions that differ from defaults;
> active branches/PRs and what they're for; known gaps with links. Derive it
> by listing and skimming — do not read whole files. Every later task (and
> every subagent) starts by reading the index, not by re-exploring; hand
> subagents only their slice. Refresh only when the tree changed. If a fact
> in the index contradicts the repo, the repo wins — fix the index in the
> same commit.
