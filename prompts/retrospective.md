# Retrospective (project pause)

> Pause and reflect on [PROJECT / SESSION] using the actual artifacts in
> [REPO OR WORKSPACE PATH]. Do not invent history.
>
> Produce:
> 1. **Keep doing:** the practices that worked, each backed by a file, command,
>    result, or user decision.
> 2. **What broke:** symptom -> root cause -> rule adopted. If a rule belongs in
>    `agent/ERRORS.md`, draft the exact entry and cite the evidence.
> 3. **What's quietly wrong:** risks nobody hit yet. If a gap belongs in
>    `agent/GAPS.md`, draft the exact entry, owner, and closing action.
> 4. **Reversible improvements:** process, settings, prompts, or harness changes
>    that improve quality, speed, or token spend. Implement only changes that are
>    reversible, local, and inside [ALLOWED PATHS].
> 5. **Verification gate:** re-run the relevant local gate after any change and
>    report exact command output. If no gate exists, state the missing gate as a
>    gap instead of claiming success.
>
> Never change credentials, network/security settings, production state, RDP,
> Defender, billing, or irreversible external systems from a retrospective. Route
> those to a human approval item with the evidence needed to decide.
